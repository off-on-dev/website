backstory

The Echo Server is misbehaving. Both environments seem to be down, and messages are silent. Your mission: investigate the Argo CD configuration and restore proper multi-environment delivery.

How to Play

  1. 1Open in GitHub Codespaces (opens in new tab). The devcontainer is pre-configured and starts automatically. GitHub will fork the repo to your account when you make your first commit.
  2. 2Prefer working locally? Clone the repo and open it in any editor that supports the Dev Containers specification (VS Code, JetBrains IDEs, and others). The devcontainer config will be detected automatically.
  3. 3Wait around 5 minutes for the Codespace to provision a Kubernetes cluster, Argo CD, and the sample app. Press Cmd+Shift+P (or Ctrl+Shift+P on Windows/Linux) and search for 'View Creation Log' to track progress.
  4. 4Open the Ports tab, find port 30100 (Argo CD), and log in with username 'readonly' and password 'a-super-secure-password'.
  5. 5Fix the ApplicationSet at adventures/01-echoes-lost-in-orbit/beginner/manifests/appset.yaml, then apply it: kubectl apply -n argocd -f adventures/01-echoes-lost-in-orbit/beginner/manifests/appset.yaml
  6. 6Run the smoke test at adventures/01-echoes-lost-in-orbit/beginner/smoke-test.sh to verify your solution locally.
  7. 7Commit and push your changes to your fork, then trigger the verification workflow on GitHub Actions to officially claim completion.
  8. 8Share your solutions and questions in the challenge thread (opens in new tab) on community.offon.dev.

Need step-by-step verification instructions? Read the Verification Guide (opens in new tab).

Objective

  • See two distinct Applications in the Argo CD dashboard (one per environment)
  • Ensure each Application deploys to its own isolated namespace
  • Make the system resilient so Argo CD automatically reverts manual cluster changes
  • Confirm that updates roll out automatically without leaving stale resources behind

Toolbox

  • kubectl (opens in new tab): Kubernetes CLI for interacting with the cluster
  • kubens (opens in new tab): fast way to switch between Kubernetes namespaces
  • k9s (opens in new tab): terminal UI for managing and inspecting your cluster
