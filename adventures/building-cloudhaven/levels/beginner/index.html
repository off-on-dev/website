backstory

The Merchant's Quarter needs essential services, but the previous Guild engineer left the OpenTofu configuration incomplete and misconfigured. Your mission: fix the issues, complete the setup, and establish proper infrastructure management for the Guild.

How to Play

  1. 1Open in GitHub Codespaces (opens in new tab). The devcontainer is pre-configured and starts automatically. GitHub will fork the repo to your account when you make your first commit.
  2. 2Prefer working locally? Clone the repo and open it in any editor that supports the Dev Containers specification (VS Code, JetBrains IDEs, and others). The devcontainer config will be detected automatically.
  3. 3Wait ~2 minutes for the environment to initialize.
  4. 4Open the Ports tab, find the GCP API Mock at port 30104. Use it to explore the mock cloud resources created by your configuration.
  5. 5All OpenTofu files are in adventures/02-building-cloudhaven/beginner/. Run grep -r "TODO" . to find everything left by the previous engineer. Files to review: main.tf, state.tf, variables.tf, merchants.tf, audit.tf, outputs.tf.
  6. 6After fixing the TODOs, run tofu apply. If you changed the backend configuration, run tofu init -migrate-state first.
  7. 7Run the smoke test: ./smoke-test.sh
  8. 8Commit and push your changes to your fork, then trigger the verification workflow on GitHub Actions to officially claim completion.
  9. 9Share your solutions and questions in the challenge thread (opens in new tab) on community.offon.dev.

Need step-by-step verification instructions? Read the Verification Guide (opens in new tab).

Objective

  • Provision storage vaults and ledger databases for each district dynamically
  • Deploy the audit database only when there is more than one district
  • Store state remotely in a GCS backend following best practices so the Guild can collaborate
  • Resolve all TODOs in the code and successfully run tofu apply

Toolbox

  • tofu (opens in new tab): OpenTofu CLI for infrastructure provisioning
  • gcp-api-mock (opens in new tab): mock GCP API running locally to simulate cloud resources without real cloud costs (Cloud Storage and Cloud SQL only)
